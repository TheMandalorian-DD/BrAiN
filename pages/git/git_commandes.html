Configuration global

* A faire en NON-administrateur dans le powershell(windows) ou terminal linux.

Votre configuration est alors enregistrée dans le fichier ~/.gitconfig.

git config --global --list pour voir l'état de la configuration.

Choisir son pseudo : git config --global user.name "Nom"
Choisir son email : git config --global user.email "Email"
Pour avoir de jolies couleurs : git config --global color.ui true
git config --global color.diff auto
git config --global color.status auto
git config --global color.branch auto
Choisir son éditeur : git config --global core.editor nano
Changer le nom d'une commande : git config --global alias.s status
Ca veut dire que si je fais git s ca revient a faire git status
On peut supprimer l'alias en tapant :
git config --global --unset alias.s
Indiquer un proxy : git config --global http.proxy http://proxy.vinci-melun.org:3128

Ignorer des fichiers de mon projet

-> Créer un fichier .gitignore touch .gitignore
->Sur chaque ligne de ce fichier on met les fichiers ou dossier qu'on souhaite ignorer.
Ex :
*.tmp -> Tous les fichiers qui ce terminent pas .tmp
fichier3.php -> le fichier3.php sera ignorer.
pages/* -> Sera ignorer tous ceux qui est à l'intérieur du dossier pages.

2
git status Pour voir l'état du dossier. (En rouge les fichiers non tracker
ils peuvent peuvent donc pas être mis dans le commit).

3
git add nomfichier.extenssion Pour tracker le fichier afin qu'il soit pris en compte lors du prochain commit.
git add --all ou git add * Pour tout ajouter.

4
git commit Pour envoyer enregistrer l'état actuelle du projet
l'éditeur de texte s'ouvrira pour nous permettre de nommer le commit
--- ou ---
git commit -m "Nom du commit" pour le faire directement
git commit -a -m "Nom du commit" pour ajouter les fichiers et commit directement

git commit --amend Si l'on fait une modification et qu'on souhaite l'enregistrer dans le commit précédent.
l'éditeur de texte s'ouvrira pour nous permettre de renommer le commit si l'on veut
--- ou ---
git commit --amend --no-edit Pour garder le nom actuelle du commit.

5
git log
 Permet s'afficher les dernier commit qui ont été fait dans mon projet.
git log --oneline récupère les informations sur une ligne.
git log --graph voir les commits graphiquement dans le terminal
git log --oneline --graph peut se faire.
git log -n 2 pour voir les deux dernières commit.
git log -p fichier2.html pour voir tous les commit qui concerne le fichier2.html.

git diff
 Prend tous les fichiers qui ont été modifier depuis le dernier
commit et nous montre les modifications.
git show clécommit
 Pour voir les modifications apporté sur le commit clécommit

Annuler un commit
git rest --hard clécommit Permet de revenir à l'état de clécommit et d'annuler
toutes les modifications ayant été faites après.
git rest --soft clécommit Permet de revenir à l'état de clécommit en gardant
toutes les modifications ayant été faites après.

git rest cléCommit Revenir à l'état du commit de notre choix mais en gardant
les modifications des commits qui ont été apportés depuis.
Les autres comit ayant eu lieu après seront supprimé
git reset HEAD^ Pour revenir un commit en arrière.
git reset HEAD^^^ Pour revenir trois commit en arrière.
git reset HEAD^ --hard Pour revenir un commit en arrière en supprimer toutes les modifications qui ont été apportés.
git rest HEAD nomFichier Pour un untracker un fichier.
git checkout cléducommit
 Voir l'état du commit tel qu'il était avant sans perdre les modifications faites depuis.
On ne peut qu'observer sans faire de modification.
git checkout cléducommit nomFichier
 Mon dépot sera modifier et le fichier reviendra a son état lors du commit cléCommit.
git log --oneline pour voir la clé.
Pour récupérer le fichier actuel il faudra faire git checkout master nomFichier

Création de branche
git branch
 Pour lister les branches
La branche master est la branche principale
* Nous permet de savoir dans qu'elle branche on se situe.
git branch -v
 Pour lister les branches mais également afficher les
derniers commit réaliser au niveau de chaque branches.
git branch nomBranche
 Créer une nouvelle branche afin de travailler sur un domaine précis de notre projet. Les modification sur cette branche ne toucherons pas les autres branches.
git checkout -b nomBranche
 Permet de créer la branche nomBranche et de se rendre immédiatement dessus.
git checkout nomBranche
 Permet de sauter d'une une autre branche.

Fusion des branches
git merge nomBranche
 Il faut se placer dans la branche principale. git checkout master
La branche nommer sera fusionner avec la branche principale
git log Pour voir si tous s'est bien passé.
Si il y a eureur de conflit il faut se rendre dans le fichier et supprimer
manuellement les lignes qu'on ne souahaite pas garder.
git rebase nomBranche
 Il faut se placer dans la branche qui va fusionner avec la principale.
La branche nommer sera fusionner avec la branche principale
Ensuite il faut aller dans la branche principale git merge nomBranche pour fusionner.
git branch -d nomBranche
 Supprime la branche Une fois qu'elle a été fusionner.
git branch -D nomBranche
 Supprime la branche même si elle a pas été fusionner.
